# 通用竞品分析框架

本模块定义分析一个竞品的通用框架，不绑定任何特定竞品或行业。
主 SKILL.md 在每次分析时通过 Read 加载本文件做参考。

---

## 1. 信息采集维度

### 1.1 产品能力
| 子维度 | 典型问题 | 采集方式 |
|--------|---------|---------|
| 产品形态 | 产品类型（SaaS/开源/一体机/平台）？核心功能？ | 官网/文档 |
| 目标用户 | 个人/企业/开发者？规模段？典型客户画像？ | 官网/案例 |
| 技术栈 | 模型/框架/语言/部署方式？ | GitHub/技术博客 |
| 发布历史 | 最近版本？更新频率？Roadmap？ | Release Notes/Changelog |

### 1.2 市场定位
| 子维度 | 典型问题 |
|--------|---------|
| 品牌叙事 | 他们的 Slogan？怎么描述自己？Key Value Prop？ |
| 目标市场 | 地域？行业？客户规模？ |
| 差异化声称 | 他们说比谁好在哪？怎么差异化？ |

### 1.3 技术架构
| 子维度 | 典型问题 |
|--------|---------|
| 模型/引擎 | 自研模型？基于开源？闭源？ |
| 部署方式 | 云端？私有化？混合？边缘？ |
| 开放性 | 开源协议？API 开放程度？可扩展性？ |
| 技术壁垒 | 核心技术（训推一体/推理优化/Agent编排）？ |

### 1.4 商业模式
| 子维度 | 典型问题 |
|--------|---------|
| 定价策略 | 免费/订阅/按量/买断？价格水平？ |
| 营收模型 | 主要收入来源？付费转化路径？ |
| 客户类型 | 付费客户画像？客单价？客户集中度？ |
| 融资情况 | 轮次？金额？估值？投资人背景？ |

### 1.5 生态与合作
| 子维度 | 典型问题 |
|--------|---------|
| 合作伙伴 | 技术合作？渠道合作？集成生态？ |
| 开发者社区 | 社区规模？活跃度？贡献者数量？ |
| 平台集成 | 支持的第三方集成？生态适配范围？ |

### 1.6 叙事与认知
| 子维度 | 典型问题 |
|--------|---------|
| 媒体报道 | 主流媒体如何报道？关注点？ |
| 行业评测 | 评测报告？分析师评价？客户证言？ |
| 竞品提及 | 他们提及昆仑元/同类产品的方式？ |

### 1.7 发展速度
| 子维度 | 典型问题 |
|--------|---------|
| 发布节奏 | Release 频率？重大更新间隔？ |

### 1.8 竞争动作
| 子维度 | 典型问题 |
|--------|---------|
| 近期发布 | 新产品？新功能？新定价？新商业模式？ |

## 2. 分析输出结构

每次分析的输出遵循以下结构（Markdown 格式）：

```markdown
## {竞品名称} — 分析摘要（{日期}）

### 能力画像
- **产品重叠度**: 高/中/低（对昆仑元的产品功能重叠）
- **技术相对水平**: 持平/领先/落后
- **市场关系**: 直接竞争/互补/无关/潜在威胁
- **威胁向量**: [叙事竞争, 定价竞争, 人才竞争, 客户争夺, 技术追赶]
- **总体态势**: 上升/稳定/下降

### 关键信号
| 类型 | 描述 | 影响 | 置信度 |
|------|------|------|--------|
| release | v2.2 发布，叙事转向 Token 工厂 | 中 | 高 |
| pricing | 开源免费，仅收 GPU 资源费 | 高 | 高 |
| partner | 与 xx 达成合作，进入 xx 领域 | 低 | 中 |

### 变化摘要
- **相对于上次跟踪**：版本 2.1 → 2.2，叙事从 GPU 调度转向 Token 工厂
- **威胁评分变化**: ↑ 5 分（上升/不变/下降）
- **新发现**: [30 天内新增的关键信号]
- **过期信息**: [上次提到但已被证伪或不再相关的内容]

### 推荐行动
- [ ] 重点关注：其向训推一体或企业版演进的信号
- [ ] 考虑应对：定价模型对比更新
- [ ] 持续观察：社区活跃度变化趋势
```

## 3. 分析原则

1. **基于证据，不推测**：每个判断附带来源引用（URL/文件名）
2. **区分事实与解读**：事实（"v2.2 发布"）和解读（"叙事转向 Token 工厂"）明确分离
3. **关注变化，不重复已知**：每次分析聚焦"上次跟踪以来有什么变化"
4. **不确定要标注**：模糊匹配标注置信度（高/中/低），不猜测
5. **与昆仑元对标**：所有判断最终对照昆仑元的产品/定位/能力做参照
