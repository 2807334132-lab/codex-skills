# 威胁评分方法

本模块定义竞品威胁评分框架。评分由主 SKILL.md 在工作流中计算，
不在模块中写死任何竞品数据。

---

## 1. 评分模型

### 六维度加权评分（总分 0-100）

| 维度 | 权重 | 评分区间 | 评分逻辑 | 判断依据示例 |
|------|------|---------|---------|-------------|
| **产品重叠度** | 25% | 0-100 | 功能匹配度 + 目标客户重合度 + 定位重合度 | 0=互补品, 25=上下游, 50=部分重叠, 75=直接竞争, 100=完全重合 |
| **市场影响力** | 20% | 0-100 | 品牌认知度 + 媒体覆盖 + 投资者认知 | 0=无人知晓, 25=小众, 50=行业知晓, 75=广泛认知, 100=家喻户晓 |
| **技术实力** | 20% | 0-100 | 技术深度 + 研发速度 + 架构先进性 | 0=基础, 25=跟随, 50=持平, 75=领先, 100=碾压 |
| **商业模式威胁** | 15% | 0-100 | 定价策略竞争 + GTM 路径重叠 + 客户争夺 | 0=无害, 25=轻微, 50=中度, 75=显著, 100=直接威胁 |
| **生态成熟度** | 10% | 0-100 | 社区规模 + 合作伙伴网络 + 集成数 | 0=无, 25=初创, 50=成长, 75=成熟, 100=垄断级 |
| **叙事竞争力** | 10% | 0-100 | 品牌故事对昆仑元叙事的侵蚀程度 | 0=无关联, 25=不同赛道, 50=松散关联, 75=叙事竞争, 100=叙事重合 |

### 计算公式

```
最终威胁评分 = Σ(维度分 × 权重)

例如：
  产品重叠度: 75 × 0.25 = 18.75
  市场影响力: 60 × 0.20 = 12.00
  技术实力:    50 × 0.20 = 10.00
  商业模式:    40 × 0.15 =  6.00
  生态成熟度:  35 × 0.10 =  3.50
  叙事竞争力:  55 × 0.10 =  5.50
  ─────────────────────────────
  总分:                   55.75
```

## 2. 优先级分配规则

### 自动分级

```
从跟踪日志读取所有竞品威胁评分 →
按威胁评分从高到低排序 →
自动分配跟踪优先级：
```

| 排名 | 优先级 | 跟踪频率 | 说明 |
|------|--------|---------|------|
| Top 2 | P0 核心 | 每周跟踪 | 最直接的竞争威胁，需要连续监控 |
| Next 3-4 | P1 重要 | 双周跟踪 | 正在成长的竞争力量 |
| 其余已评分 | P2 观察 | 月度跟踪 | 值得关注但非紧急 |
| 未评分 | P3 背景 | 季度跟踪 | 新发现竞品，信息不足，先标记后评估 |

### 规则说明

- **P0/P1 数量不是固定的**：竞品池增加时 P0 保持 Top 2，P1 保持 Top 5（排除 P0 后）
- **新竞品自动进入 P3**：刚 `track` 或 `init` 导入的竞品从 P3 开始
- **评分后自动升级**：`check` 后更新评分，触发优先级重新分配
- **降级机制**：如果某个竞品连续 3 次 check 威胁评分下降超过 10 分，自动降一级
- **手工覆盖**：可通过命令 `/competitor priority <name> <P0|P1|P2|P3>` 覆盖自动分配

## 3. 评分变化追踪

每次 `check` 或 `report` 执行时，评分变化记录到跟踪日志中：

```markdown
### 威胁评分变化
| 日期 | 评分 | 变化 | 主要驱动因素 |
|------|------|------|-------------|
| 2026-06-22 | 55 | - | 初始基线 |
| 2026-06-29 | 62 | +7 | v2.2 发布，叙事转向 Token 工厂 |
| 2026-07-06 | 58 | -4 | 社区活跃度下降，新功能未达预期 |
```

## 4. 评分注意事项

1. **评分必须附带判断理由**，不能只给数字不给依据
2. **关注趋势而非绝对值**：单个评分有误差，趋势比绝对值更有参考价值
3. **跨竞品不可比**：不同赛道的竞品评分不具备直接可比性（如 GPU 调度平台 vs Agent 平台）
4. **定期校准**：每月报告时复核所有 P0/P1 竞品评分，避免评分漂移
5. **不确定性标记**：信息不足时标记为 "待评估（信息不足）"，不强行评分
